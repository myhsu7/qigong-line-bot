# Qigong LINE Bot - Roadmap v3.0

這份 Roadmap 整理了系統穩定後的下一步發展方向，將重點放在「提升數據洞察」、「群組營運效率」以及「使用者體驗升級」三個核心面向。

---

## 🚀 短期計畫 (Short-term)
**目標：將 Phase 1 的「功法分析」正式產品化，並強化 Admin Dashboard。**

1.  **「我的功法配比」Bot 指令**
    *   **說明：** 將 `method_stats_guide.md` 裡的 SQL 報表邏輯寫入 Node.js。
    *   **UX：** 學員在 1-on-1 聊天室輸入 `📈 我的功法分析`，Bot 自動回覆最近 30 天/90 天的功法圓餅圖數據與文字點評（如：你本月最常練龜壽功，佔 60%）。
2.  **Admin Dashboard V2 (視覺化升級)**
    *   **說明：** 將儀表板擴充。
    *   **功能：** 
        *   新增 `年度 (Annual)` 總覽的切換。
        *   加入全體學員的「功法偏好分布圖」（Pie chart / Bar chart）。
3.  **群組里程碑自動廣播 (Milestone Shoutouts)**
    *   **說明：** 當有人達到 100 天（百日築基）等重大成就時，8 PM 的每日提醒會自動加入該名學員的恭喜文案，提升群組凝聚力。

---

## 🛤️ 中期計畫 (Medium-term)
**目標：優化打卡流程，解決文字辨識誤判，並提升管理效率。**

1.  **結構化打卡系統 (Structured Check-in)**
    *   **說明：** 放棄完全自由填寫的 note。
    *   **UX：** 點擊 `✅ Check-In` 後，使用 LINE Flex Message 的選單（或 LIFF App），讓學員用按鈕直接勾選今天練的功法（可複選），最後再附上自由備註。
    *   **效益：** 統計準確率達到 100%，不再需要維護「同義詞字典」。
2.  **Admin Dashboard 個人透視頁 (Drill-down)**
    *   **說明：** 在排行榜上點擊學員名字，可進入該學員專屬頁面。
    *   **功能：** 查看他的長期打卡折線圖、功法轉換趨勢，並支援將其報表匯出成 CSV 檔。
3.  **自動化群組報表 (Automated Reports)**
    *   **說明：** 每週一/每月一號，自動生成精美的群組戰報（例如：本月全體共打卡 500 次，最受歡迎功法是回春功），推播到群組。

---

## 🔭 長期計畫 (Long-term)
**目標：系統化、高可用性，並建立完整的社群生態。**

1.  **落實 HA 與備份演練 (Infrastructure Hardening)**
    *   **說明：** 根據 `backupandrestoreplan.md`，實際建立遠端 PostgreSQL Standby 備援機。
    *   **演練：** 建立每月的還原測試 SOP，確保無懼機器損壞。
2.  **節氣/時令專題活動系統 (Seasonal Campaigns Module)**
    *   **說明：** 將夏練三伏、冬練三九的邏輯抽象化，變成後台可以自由設定的「期間限定任務」。
    *   **功能：** 管理者可在介面上設定任務名稱、起訖時間、獎勵徽章與過關條件。
3.  **AI 輔助評語 (AI Companion)**
    *   **說明：** 串接 OpenAI / Gemini API。
    *   **功能：** 根據學員每天輸入的備註心得，給予個人化的養生建議與鼓勵（確保 Prompt 限制在不涉及醫療診斷的練功陪伴）。
