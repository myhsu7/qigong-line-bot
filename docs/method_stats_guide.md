# 功法統計與分析指南 (Phase 1)

這份指南說明如何使用自由填寫的打卡筆記 (`checkin_logs.note`)，透過關鍵字比對的方式來分析學員的功法練習分布。

---

## 1. 功法字典維護規範 (Method Dictionary)

為了讓統計更精確，我們必須定義每種功法會被提及的「同義詞」或「簡稱」。  
目前在 SQL 中我們使用的是 V1 版的字典：

| 標準功法名稱 | 同義詞 / 可能輸入的字眼 |
| :--- | :--- |
| **大雁功** | `大雁功` |
| **回春功** | `回春功` |
| **龜壽功** | `龜壽功` |
| **正陽功** | `正陽功` |
| **神奇晃海功** | `神奇晃海功`, `晃海功`, `晃海` |
| **和氣舒壓法** | `和氣舒壓法`, `和氣`, `舒壓法` |
| **蓮花** | `蓮花`, `蓮花功` |

**💡 維護建議：**
- **新增功法/同義詞**：只需直接在 SQL 查詢的 `VALUES` 區塊中添加。
- **避免誤判**：不要設定過短或過於常見的詞，例如如果把「氣」當作關鍵字，幾乎所有的筆記都會被命中。
- **定期人工抽樣**：每月隨機從 `checkin_logs` 抽 20 筆含有「和氣」的筆記，人工檢查是否真的在講「和氣舒壓法」，若發現常有誤判，再把字典修正得更精確。

---

## 2. 分析報表 SQL 說明

我們將報表存放於 `reports/sql/` 目錄下：

1. `user_method_trend_monthly.sql` (假設您已經有草稿，同週/季邏輯)
2. `user_method_trend_weekly.sql`
3. `user_method_trend_quarterly.sql`

**使用方法：**
您可以使用 pgAdmin、DBeaver 或是命令列連進資料庫，執行這些 SQL。執行前請將 SQL 末端的 `:uid` 替換成要查詢的學員 `line_user_id`。

---

## 3. 「我的功法比例摘要」訊息模板 (UX 模板)

當你跑出學員的功法比例（例如最近 30 天）後，可以用以下文案模板直接貼給學員，作為個人化的練功 review。

**【文案模板】**
```text
📊 你的近期練功成分分析 (最近 30 天)

哈囉！整理了你最近 30 天的打卡紀錄，來看看你的練功配方吧：

🧘 主要修練功法：
1. {功法A}：佔 {X}% (共 {Y} 天)
2. {功法B}：佔 {X}% (共 {Y} 天)
3. {功法C}：佔 {X}% (共 {Y} 天)

💡 小點評：
你最近花最多的時間在「{功法A}」，非常穩定！下個月也許可以試著加入一點「{推薦功法}」來平衡身心喔！
```

**【實際範例】**
```text
📊 你的近期練功成分分析 (最近 30 天)

哈囉！整理了你最近 30 天的打卡紀錄，來看看你的練功配方吧：

🧘 主要修練功法：
1. 龜壽功：佔 60% (共 15 天)
2. 回春功：佔 30% (共 8 天)
3. 神奇晃海功：佔 10% (共 2 天)

💡 小點評：
你最近花最多的時間在「龜壽功」，非常穩定！下個月也許可以試著加入一點「大雁功」來平衡身心喔！
```
