# 白雁氣功常用功法別名字典

這份文件用來整理白雁氣功學員在日常打卡時常用的功法寫法，作為 `method-analysis` 的 alias dictionary 參考。

## 說明

- 目前 `method-analysis` Phase 1 仍採用 **自由文字 + 關鍵字比對**。
- 因此只要學員輸入的寫法沒有被收錄進 alias dictionary，就可能漏算。
- 例如：
  - `大雁初高` 如果沒有被歸類到 `大雁功`，就不會命中。
  - `正陽晨功` 如果沒有被歸類到 `正陽功`，也不會命中。

## 建議不要直接納入的模糊寫法

以下字眼雖然可能與功法有關，但過於模糊，直接納入會提高誤判風險：

- `晨功`
- `初高`
- `高級`
- `舒壓`
- `和氣`（需抽樣確認後再決定是否保留）

## 建議資料庫驗證方式

由於我目前無法直接連到你正式的 PostgreSQL runtime database，下面提供一組可在 Ubuntu server / pgAdmin / DBeaver 上直接執行的 SQL，用來把目前學員常用的打卡字樣列出來，再人工 grouping。

### 1. 列出最近常見的完整打卡 note

```sql
SELECT
  note,
  COUNT(*) AS freq
FROM checkin_logs
WHERE note IS NOT NULL
  AND note <> ''
GROUP BY note
ORDER BY freq DESC, note ASC
LIMIT 200;
```

用途：
- 先看大家最常直接輸入哪些完整句子
- 特別適合抓出像 `正陽晨功 大雁初高` 這種固定搭配

### 2. 只篩白雁氣功相關關鍵字的 note

```sql
SELECT
  note,
  COUNT(*) AS freq
FROM checkin_logs
WHERE note ILIKE '%大雁%'
   OR note ILIKE '%回春%'
   OR note ILIKE '%龜壽%'
   OR note ILIKE '%正陽%'
   OR note ILIKE '%晃海%'
   OR note ILIKE '%和氣%'
   OR note ILIKE '%舒壓%'
   OR note ILIKE '%蓮花%'
GROUP BY note
ORDER BY freq DESC, note ASC;
```

用途：
- 快速聚焦在功法相關打卡內容
- 幫助人工檢查別名是否需要擴充

### 3. 檢查某個候選別名是否應該納入

以 `大雁初高` 為例：

```sql
SELECT
  line_user_id,
  created_at AT TIME ZONE 'Asia/Taipei' AS local_time,
  note
FROM checkin_logs
WHERE note ILIKE '%大雁初高%'
ORDER BY created_at DESC
LIMIT 50;
```

用途：
- 確認這個字眼在上下文中是不是穩定指向同一種功法
- 若 95% 以上都指向同一功法，就可以放心納入 alias dictionary

### 4. 建議人工 grouping 輸出表

你可以把查詢結果整理成以下格式：

| 原始打卡字樣 | 建議歸類 | 是否納入 | 備註 |
| :--- | :--- | :--- | :--- |
| 大雁初高 | 大雁功 | 是 | 已確認常用 |
| 正陽晨功 | 正陽功 | 是 | 已確認常用 |
| 和氣 | 和氣舒壓法 | 待確認 | 需抽樣避免誤判 |
